> 請說明檔案夾 node_modules 的功能                                              

● node_modules 是 Node.js 專案的套件依賴儲存目錄。

  當你執行 npm install 時，npm 會讀取 package.json
  裡列出的依賴套件，並將它們下載、解壓縮到 node_modules 資料夾中。你的程式碼（如
   make-pdf.js）用 require('puppeteer-core') 時，Node.js
  就從這個資料夾找到並載入對應的模組。

  在這個專案裡，node_modules 裡最主要的套件是 puppeteer-core——它提供控制 Chrome
  瀏覽器的 API，讓 make-pdf.js 能夠開啟 catalog.html 並輸出 PDF。

  幾個實務要點：

  - node_modules 通常很大（幾十到幾百 MB），不應該上傳到 Git，應在 .gitignore
  中排除
  - 如果這個資料夾被刪除，只要有 package.json，執行 npm install 即可重新建立
  - 它的內容完全由 package.json（或更精確的
  package-lock.json）決定，本身不需要手動編輯

> 上傳雲端網站時，需要上傳 node_modules 檔案夾嗎？                              

● 不需要。

  node_modules 只是 make-pdf.js（PDF 產生腳本）在你本機執行時需要的工具。

  上傳網站時，你只需要上傳：

  Butterfly Effect/
  ├── index.html        ← 網頁主體
  ├── sketch.js         ← p5.js 主程式
  ├── vert.glsl         ← 頂點著色器
  ├── frag.glsl         ← 片段著色器
  ├── lod.js            ← LOD 控制器
  ├── music.js          ← 音樂管理器
  └── Music/            ← 音樂檔案資料夾

  不需要上傳：

  node_modules/         ← 只有本機產生 PDF 時才用
  make-pdf.js           ← PDF 產生腳本，網頁不需要
  catalog.html          ← PDF 的原始稿，網頁不需要
  EXHIBITION-CATALOG.*  ← 展覽手冊檔案，網頁不需要
  package.json          ← npm 設定，網頁不需要

  node_modules 裡的 puppeteer-core 是用來控制本機 Chrome 產生 PDF
  的，瀏覽器在雲端伺服器上直接載入 index.html 時完全不需要它。
